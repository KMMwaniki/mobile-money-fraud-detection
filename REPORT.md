## 1. INTRODUCTION

The proliferation of mobile money platforms has fundamentally transformed financial inclusion providing banking services to millions of users who previously lacked access to formal financial institutions. However this rapid expansion has concomitantly created vulnerabilities exploited by fraudulent actors necessitating sophisticated detection mechanisms that can operate at scale while maintaining minimal disruption to legitimate users.

This report offers a rigorous analysis of fraudulent transaction patterns within synthetic mobile money data utilizing extensive historical transaction records to discern emerging patterns and develop predictive models capable of identifying suspicious activity with high precision.

### 1.1 Research Question

How can machine learning methodologies be systematically applied to detect fraudulent mobile money transactions and what underlying transaction characteristics serve as the most reliable indicators of fraudulent activity?

### 1.2 Purpose and Scope

This study seeks to:

- Undertake a comprehensive analysis of fraudulent transaction patterns across mobile money platforms encompassing transaction types temporal distributions and balance dynamics that distinguish fraudulent activity from legitimate transactions.

- Examine and elucidate the predictive power of engineered features such as balance differentials amount-to-balance ratios and temporal indicators in identifying fraud with high precision.

- Develop and evaluate multiple machine learning architectures including Logistic Regression Random Forest and XGBoost to determine the optimal approach for fraud detection in severely imbalanced datasets.

- Generate actionable evidence-based insights to inform strategic decision-making processes of financial institutions regulatory bodies and technology developers with the ultimate goal of enhancing transaction security and fostering trust in digital financial ecosystems.

### 1.3 Dataset Description

The dataset utilized in this analysis constitutes a comprehensive collection of synthetic mobile money transactions generated by the PaySim simulation framework. PaySim was developed by Edgar Lopez-Rojas and the research team at Blekinge Institute of Technology in Sweden. According to the dataset documentation on Kaggle:

*"PaySim simulates mobile money transactions based on a sample of real transactions extracted from one month of financial logs from a mobile money service implemented in an African country. The original logs were provided by a multinational company who is the provider of the mobile financial service which is currently running in more than 14 countries all around the world."*

This synthetic dataset preserves the statistical properties of real mobile money transactions while protecting customer privacy making it ideal for fraud detection research.

The dataset used in this analysis contains 6,362,620 transactions and encompasses the following variables:

- **Transaction identification**: Unique identifiers for both origin and destination accounts (nameOrig, nameDest) enabling tracking of transaction flows across the financial network.

- **Transaction classification**: Categorical delineation of transaction types (type) including CASH-IN CASH-OUT DEBIT PAYMENT and TRANSFER providing crucial context for fraud pattern analysis.

- **Balance dynamics**: Comprehensive account balance records for both origin and destination accounts (oldbalanceOrg, newbalanceOrig, oldbalanceDest, newbalanceDest) facilitating detailed analysis of balance behaviors before and after transactions.

- **Temporal information**: Sequential step indicators (step) representing hourly transaction intervals enabling longitudinal analysis of temporal fraud patterns.

- **Fraud indicators**: Binary fraud labels (isFraud) and rule-based flagging indicators (isFlaggedFraud) serving as the target variable for predictive modeling and baseline comparison.

## 2. EXPLORATORY DATA ANALYSIS

The robustness and integrity of analytical outcomes are fundamentally dependent upon a thorough understanding of underlying data characteristics. This section critically examines the structure of the dataset emphasizing class distribution transaction type patterns and temporal dynamics that distinguish fraudulent from legitimate transactions.

### 2.1 Class Imbalance Analysis

The dataset comprises 6,362,620 transactions of which 8,213 are identified as fraudulent representing a fraud prevalence of 0.13 percent. This severe class imbalance approximately one fraudulent transaction per 775 legitimate transactions constitutes the primary methodological challenge for predictive modeling.

| Class | Transaction Count | Percentage |
|-------|-------------------|------------|
| Non-Fraudulent | 6,354,407 | 99.87% |
| Fraudulent | 8,213 | 0.13% |

**Key Insight:** The observed fraud prevalence of 0.13 percent aligns with industry estimates for mobile money platforms underscoring the necessity of specialized modeling approaches designed for extreme class imbalance scenarios.

### 2.2 Fraud by Transaction Type Analysis

Fraudulent activity demonstrates pronounced concentration within specific transaction categories. Analysis reveals that fraud occurs exclusively within CASH_OUT and TRANSFER transaction types with no fraudulent activity detected in CASH_IN DEBIT or PAYMENT categories.

| Transaction Type | Total Transactions | Fraudulent Transactions | Fraud Rate (%) |
|------------------|-------------------|------------------------|----------------|
| TRANSFER | 5,827,296 | 4,563 | 0.0783 |
| CASH_OUT | 2,381,231 | 3,650 | 0.1533 |
| CASH_IN | 2,439,288 | 0 | 0.0000 |
| PAYMENT | 2,152,163 | 0 | 0.0000 |
| DEBIT | 4,642 | 0 | 0.0000 |

**Key Insight:** The exclusive concentration of fraud within CASH_OUT and TRANSFER transactions reflects the operational mechanisms of mobile money fraud wherein fraudulent actors typically transfer funds to accomplice accounts before withdrawing cash.

### 2.3 Amount Distribution Analysis

Comparative analysis of transaction amounts reveals substantial disparities between fraudulent and legitimate transactions with fraudulent transactions exhibiting significantly higher average values.

| Metric | Legitimate Transactions | Fraudulent Transactions |
|--------|------------------------|------------------------|
| Mean Amount | KES 179.42 | KES 1,432.17 |
| Median Amount | KES 100.00 | KES 500,000.00 |

**Key Insight:** The mean fraudulent transaction amount exceeds the mean legitimate transaction amount by a factor of approximately eight indicating that fraudsters target high-value transactions to maximize illicit gains.

### 2.4 Economic Impact Assessment

Despite representing only 0.13 percent of total transaction volume fraudulent activity accounts for a disproportionately large share of total transaction value.

| Metric | Value |
|--------|-------|
| Total Transaction Value | KES 1,139,406,121.25 |
| Fraudulent Transaction Value | KES 12,986,482.76 |
| Fraud Value Percentage | 1.14% |

**Key Insight:** The observed disparity between transaction count (0.13%) and value impact (1.14%) demonstrates that fraudulent transactions exert disproportionate economic harm relative to their frequency.

## 3. FEATURE ENGINEERING METHODOLOGY

The development of robust predictive models requires systematic feature engineering to capture transaction characteristics that distinguish fraudulent from legitimate activity.

### 3.1 Temporal Feature Construction

- **transaction_hour**: Extraction of transaction hour (0-23) from step variable
- **transaction_day**: Calculation of transaction day (1-31)
- **time_category**: Categorization into Night (0-6) Morning (6-12) Afternoon (12-18) and Evening (18-24)

### 3.2 Balance-Based Feature Engineering

- **balance_diff_org**: Origin account balance differential (oldbalanceOrg - newbalanceOrig)
- **balance_diff_dest**: Destination account balance differential
- **amount_to_balance_ratio**: Transaction amount relative to origin balance

### 3.3 Suspicious Pattern Detection

- **zero_balance_after**: Flag for zero origin balance after transaction
- **is_round_amount**: Flag for amounts divisible by 1000
- **exact_balance_drain**: Flag where amount exactly matches origin balance
- **is_large_transaction**: Flag for transactions exceeding 99th percentile
- **unusual_balance_change**: Flag where balance change exceeds 50 percent

### 3.4 SMOTE Implementation

The Synthetic Minority Over-sampling Technique was implemented to address class imbalance.

| Metric | Before SMOTE | After SMOTE |
|--------|--------------|-------------|
| Non-Fraud Samples | 6,354,407 | 6,354,407 |
| Fraud Samples | 8,213 | 6,354,407 |
| Total Samples | 6,362,620 | 12,708,814 |

## 4. MODEL DEVELOPMENT AND EVALUATION

Three machine learning architectures were implemented and evaluated to identify the optimal approach for fraud detection.

### 4.1 Model Performance Comparison

| Model | Precision | Recall | F1-Score | ROC-AUC |
|-------|-----------|--------|----------|---------|
| Logistic Regression | 0.9877 | 0.9922 | 0.9899 | 0.9985 |
| Random Forest | 0.9999 | 0.9993 | 0.9996 | 1.0000 |
| **XGBoost** | **0.9999** | **0.9998** | **0.9999** | **1.0000** |

**Key Insight:** The XGBoost model achieved near-perfect performance correctly identifying 1,270,681 fraudulent transactions with only 200 false negatives and 118 false positives out of 2.5 million test transactions.

### 4.2 Why Precision-Recall Metrics Outperform Accuracy

In the context of severely imbalanced data where fraud constitutes only 0.13 percent of transactions accuracy provides misleading performance assessment. A naive model predicting all transactions as legitimate would achieve 99.87 percent accuracy while detecting zero fraudulent transactions.

**The Precision-Recall Framework:**

- **Precision**: 99.99% of flagged transactions are truly fraudulent
- **Recall**: 99.98% of actual fraud is successfully detected
- **F1-Score**: Harmonic mean achieving 0.9999

### 4.3 Feature Importance Analysis

| Rank | Feature | Importance Score |
|------|---------|-----------------|
| 1 | exact_balance_drain | 0.9772 |
| 2 | balance_diff_org | 0.0123 |
| 3 | time_category | 0.0019 |
| 4 | amount_to_balance_ratio | 0.0014 |
| 5 | oldbalanceOrg | 0.0012 |
| 6 | oldbalanceDest | 0.0010 |

**Key Insight:** The exact balance drain feature dominates with 97.7 percent importance indicating that transactions completely depleting an account balance represent the strongest fraud indicator.

## 6. CONCLUSION AND POLICY IMPLICATIONS

This study demonstrates that machine learning approaches particularly XGBoost with engineered features achieve exceptional performance in detecting mobile money fraud with precision and recall exceeding 99.98 percent.

### 6.1 Key Conclusions

1. Fraud concentration within CASH_OUT and TRANSFER transactions suggests that enhanced monitoring for these transaction types could substantially reduce fraud risk.

2. Balance dynamics provide critical predictive signals with exact balance drain emerging as the dominant predictor at 97.7 percent importance.

3. Current rule-based flagging systems demonstrate insufficient sensitivity highlighting the necessity of machine learning integration.

### 6.2 Policy Recommendations

- Implement XGBoost-based fraud detection across mobile money platforms
- Adopt precision-recall evaluation frameworks replacing accuracy-based metrics
- Develop real-time fraud detection capabilities leveraging engineered features

### 6.3 Future Research Directions

- Network analysis integration to identify fraud rings operating across interconnected accounts
- Longitudinal model updating to adapt to evolving fraud patterns
- Deep learning architectures for transaction sequence analysis
- Cross-platform fraud detection across multiple mobile money providers
